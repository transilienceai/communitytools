# Small-Prime / Multi-Source Factorization

## When this applies

- RSA modulus `n` with at least one prime factor `< 2^80` (small enough for trial division, ECM, or Pollard rho).
- Many RSA moduli published; some pairs may share a prime factor (batch GCD).
- Trigger keywords: "smooth", "small factor", "trial division", "ECM", "factordb", "common factor", "batch GCD".

## Technique

Several composable methods:

1. **factordb lookup.** If the modulus has been factored before (CTFs, leaked test keys), factordb knows.
2. **Trial division.** Test small primes up to ~2^24.
3. **Pollard rho.** Finds factors up to ~2^40 in seconds, ~2^60 in minutes.
4. **ECM (lenstra elliptic-curve method).** Finds factors up to ~2^80 in tens of minutes; up to 2^100 with effort.
5. **Batch GCD across many moduli.** If two RSA users picked overlapping primes (poor entropy), `gcd(n_i, n_j)` factors both. Heninger et al. 2012 found 0.5% of internet RSA keys this way.
6. **Roca (CVE-2017-15361, "Infineon").** Specific structured-prime form vulnerable to lattice attack — relevant for keys generated by Infineon TPM / smartcards 2012-2017.

## Steps

### 1. factordb

```bash
curl "http://factordb.com/api?query=<n>" | jq
```

If status is `FF` ("fully factored"), grab the factor list.

### 2. Trial division

```python
import gmpy2
for p in primes_below(10**6):
    if n % p == 0:
        print(p, n // p)
        break
```

### 3. Pollard rho (Brent variant)

```python
def rho(n):
    if n % 2 == 0: return 2
    x = 2; y = 2; c = 1; d = 1
    while d == 1:
        x = (x*x + c) % n
        y = (y*y + c) % n
        y = (y*y + c) % n
        d = gmpy2.gcd(abs(x - y), n)
    return d if d != n else rho(n)  # retry with different c on cycle

f = rho(n)
```

### 4. ECM via `gmpy2` or `sympy`

```python
import sympy
factors = sympy.factorint(n, limit=10**8)
```

For larger: `yafu`, `cado-nfs`, or `gmp-ecm` CLI.

### 5. Batch GCD

```python
moduli = [n1, n2, n3, ...]
from itertools import combinations
import math
for n_i, n_j in combinations(moduli, 2):
    g = math.gcd(n_i, n_j)
    if 1 < g < n_i:
        print("shared factor:", g)
```

For thousands of moduli, use product-tree GCD (`O(N log^2 N)` instead of `O(N^2)`).

### 6. Roca check

```python
# Infineon-style primes: p = k * M + (65537^a mod M) for fixed M

```

If fingerprint matches, run the published Roca lattice attack.

## Verifying success

- `p * q == n` (or for multi-prime, product equals `n`).
- All factors prime (`gmpy2.is_prime`).
- Decryption produces expected plaintext.

## Common pitfalls

- **Skipping factordb.** Many CTF moduli are already on factordb — always check first; saves hours.
- **Pollard rho infinite loop.** If the cycle returns `n`, restart with different `c`. Some implementations don't retry.
- **ECM tuning.** ECM's `B1` parameter trades runtime for factor-size reach. Default `B1=11000` finds ~25-digit factors; `B1=10^6` reaches 35 digits.
- **Multi-prime RSA.** A modulus with 3+ primes won't yield to single-step Fermat; chain methods (factor smallest first, recurse on cofactor).
- **Roca-vulnerable but valid-looking key.** A 2048-bit Roca key can be factored on a laptop in hours; 1024-bit in minutes. Always fingerprint TPM-generated keys.

## Tools

- `factordb.com` — first stop.
- `sympy.factorint` — pure Python, good up to ~2^60 factors.
- `yafu` — drop-in CLI for stronger factoring.
- `gmp-ecm` — best ECM implementation.
- `crocs-muni/roca` — Roca fingerprint + attack.
- Heninger's "fastgcd" — batch GCD across millions of moduli.
