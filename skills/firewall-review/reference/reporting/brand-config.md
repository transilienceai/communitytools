# Brand Config

**Reference implementation:** `fwrr.report.brand.load` in [firewall-review](https://github.com/ipunithgowda/firewall-review)
**Version pin:** (see ../VERSIONS.md)

## Role in the pipeline
Called once per report render. `load()` returns the merged `BRAND` dict that `excel.py` (worksheet titling, palette fills), `pdf.py` (cover page, footer, disclaimer), and `report.html.j2` (HTML template) read keys from. No state — re-call to pick up `brand.yaml` edits between renders.

## What it does
Resolves `brand.yaml` at the repo root via `Path(__file__).resolve().parents[3] / "brand.yaml"` (the file lives at [`lib/fwrr/report/brand.py`](https://github.com/ipunithgowda/firewall-review/blob/main/lib/fwrr/report/brand.py), so `parents[3]` walks `report -> fwrr -> lib -> repo-root`). If the file exists, parses it with `yaml.safe_load`; on `yaml.YAMLError` falls back to an empty dict (silent — no warning). Merges as `{**_DEFAULTS, **overrides}` so any key the YAML omits keeps the OSS-neutral default. If `brand.yaml` does not exist, returns `dict(_DEFAULTS)` (a fresh copy).

## Inputs
- None. The only input is the optional `brand.yaml` file at the repo root, which is gitignored.

## Outputs
A flat `dict` with the keys below. Defaults shown:

| Key | Default | Used by |
|---|---|---|
| `name` | `"Firewall Review"` | All renderers |
| `tagline` | `"Audit-grade firewall ruleset review"` | PDF cover, Excel header |
| `url` | `"https://github.com/ipunithgowda/firewall-review"` | PDF footer |
| `audit_email` | `"audit@example.com"` | Document Control sheet |
| `company` | `"your-organization"` | All renderers (legal entity) |
| `address` | `""` | Document Control sheet (intentionally empty in OSS) |
| `department` | `"Security & Audit"` | Document Control sheet |
| `disclaimer_long` | (paragraph, see below) | Document Control disclaimer block |
| `notice_ip` | `""` | Document Control Notice block (IP) |
| `notice_nowarranty` | `""` | Document Control Notice block (warranty) |
| `notice_changes` | `""` | Document Control Notice block (changes) |
| `primary_navy` | `"#1B3A5C"` | Excel palette / PDF accent |
| `primary_blue` | `"#2E5F8A"` | Excel palette / PDF accent |
| `accent_red` | `"#C00000"` | Severity highlights |
| `accent_green` | `"#00B050"` | Confirmed-status highlights |

`disclaimer_long` default: `"This document is generated by Firewall Review, an open-source firewall ruleset audit tool. Findings require review by a qualified security auditor before reliance for certification or regulatory submission."`

The three `notice_*` fields are intentionally empty strings in OSS so the Document Control Notice block renders blank rows rather than leaking brand-specific IP / warranty / change-management language. Internal builds may override any of the three independently.

## Gotchas / failure modes
- **Silent YAML errors.** A malformed `brand.yaml` is caught (`except yaml.YAMLError`) and treated as no-overrides. There is no warning to the operator — a typo means the report ships with OSS defaults.
- **`parents[3]` is a hard-coded path walk.** Moving `brand.py` out of [`lib/fwrr/report/`](https://github.com/ipunithgowda/firewall-review/blob/main/lib/fwrr/report/) (or restructuring the `lib/` tree) breaks the resolver silently — `brand.yaml` will appear absent and defaults will load.
- **Hex without alpha prefix.** Palette values are 7-char `#RRGGBB`. The docstring on `_DEFAULTS` notes "excel.py re-adds it" — the `0xFF` alpha prefix for openpyxl-style ARGB conversion is the renderer's job, not this loader's.
- **`address` empty by default.** Some report layouts may render an empty footer cell when not overridden — fill it for internal builds.
- **No schema validation.** Any extra key in `brand.yaml` is merged in and exposed to renderers. Misspelled keys silently no-op (the defaults still win for the correctly-spelled key).

## When to modify
- **Re-skin for an internal build:** copy `brand.yaml.example` to `brand.yaml`, uncomment the lines for the keys you want to override, and fill in. The file is gitignored — don't commit it.
- **OSS forks:** delete `brand.yaml` to fall back fully to neutral defaults.
- **New brand field:** add to `_DEFAULTS`, document in `brand.yaml.example`, and consume in the matching renderer (`excel.py`, `pdf.py`, or `report.html.j2`).
- **Template:** `brand.yaml.example` at the repo root is the canonical re-skin reference — every key here is documented there.
